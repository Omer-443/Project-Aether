# PROJECT AETHER: MASTER CONTEXT & ARCHITECTURE

## 1. PROJECT OVERVIEW
We are building "Project Aether", a futuristic, production-ready AI system for predicting healthcare claim denials using irregular, real-world time-series data. 
This is NOT a standard LSTM or Transformer project. We are implementing a **Continuous-Time Liquid Neural CDE (Controlled Differential Equation)** based on MIT CSAIL (Liquid NNs) and Oxford/Stanford (Neural CDEs) research.

## 2. STRICT TECHNICAL RULES (DO NOT VIOLATE)
- **NO STANDARD RNNs:** Do not use `nn.LSTM`, `nn.GRU`, or standard Transformers for the time-series modeling. 
- **MUST USE `torchcde`:** The core model must use `torchcde.CDEFunc` and `torchcde.cde_integrate`.
- **Liquid Vector Field:** The vector field inside the CDE must be parameterized by a Liquid Time-Constant (LTC) layer (using `ncps` or custom ODE implementation) so the equations adapt dynamically.
- **Continuous Time:** Time $t$ must be passed as a continuous physical variable (e.g., days since epoch), not a discrete index.
- **Frontend Vibe:** The UI must be "Cyberpunk meets Clinical Cleanliness". Dark mode, deep slate/zinc backgrounds, glowing emerald/cyan accents, glassmorphism cards, and fluid Framer Motion animations. It must look like a billion-dollar Silicon Valley SaaS.

## 3. BACKEND ARCHITECTURE (FastAPI + PyTorch)
### `backend/app/core/math_engine.py`
- Implement `LiquidCDEModel(nn.Module)`.
- It must take `coeffs` (from Hermit cubic spline interpolation of the data) and integrate them using `torchcde`.
- Include a `simulate_policy_shock()` method that alters the vector field weights dynamically to prove the model adapts without retraining.

### `backend/data_pipeline/cde_formatter.py`
- Write a function `prepare_cms_for_cde(df)`.
- It must take irregularly spaced timestamps, sort them, and use `torchcde.hermite_cubic_coefficients_with_backward_differences` to generate the continuous path coefficients. This is critical for `torchcde` to work.

### `backend/app/api/routes.py`
- `POST /api/v1/predict`: Takes a JSON of irregular claim events, formats to CDE coeffs, runs inference, returns the continuous trajectory and final denial probability.
- `POST /api/v1/shock-test`: Simulates a TPA policy change and returns the model's adaptation metrics.

## 4. FRONTEND ARCHITECTURE (Next.js 14 + Tailwind + Framer)
### `frontend/app/globals.css`
- Set up a dark theme. Use CSS variables for glowing effects (e.g., `--glow-emerald: 0 0 15px rgba(16, 185, 129, 0.5)`).

### `frontend/components/ContinuousCurve.tsx`

- Use Recharts to plot the *continuous latent trajectory* generated by the backend. 
- Add Framer Motion so the curve "draws" itself smoothly when data is submitted. Show the "gaps" in data as smooth, confident curves, visually proving the CDE handles irregular time.

### `frontend/components/PolicyShockPanel.tsx`
